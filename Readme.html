In this figure, the processing of one event ek,je_{k,j}ek,j​ belonging to trace σj\sigma_jσj​ is schematised.

First of all, the labels received are only used to train the models in the framework. The event ek,je_{k,j}ek,j​ may either be the last of σj\sigma_jσj​, i.e., length of jjj, in which case the label yjy_jyj​ becomes known, or not. When an event is not the last one of its trace (see Fig. 1.a), it is used to generate a new prefix of σj\sigma_jσj​ for prefix kkk. Then, a prediction y^k,j\hat{y}_{k,j}y^​k,j​ for the new prefix can be computed using the model for prefix kkk. Receiving the last event ek,je_{k,j}ek,j​, and its label (see Fig. 1.b) enables (i) to evaluate all the predictions y^j\hat{y}_jy^​j​ that have been generated for all prefixes σj\sigma_jσj​ (evaluation before training), (ii) to update the models pomnpom_npomn​ owing to the availability of new labelled prefixes. Finally, it is possible (iii) to compute a new set of predicted labels y^k,l\hat{y}_{k,l}y^​k,l​, with l≠jl\neq jl​=j and for all the prefixes for which a label has not been yet received (train and retest).

2)  Evaluation methods: supporting example

Fig. 2 exemplifies what stated above in the context of the proposed framework, considering 3 process cases and prefix length up to 3.

Note that different versions of the same model pomk are generated along the considered timeline. In particular, a new version of pomkpom_kpomk​ is generated when a new label yjy_jyj​ for a case σj\sigma_jσj​. For instance, in the example 3 different versions of pom1pom_1pom1​ are generated. Second, new predictions for prefixes of length kkk are generated each time a new version of pomkpom_kpomk​ is available. In the example, receiving the label yc1y_{c1}yc1​ of case c1 at t8 triggers (i) the creation of a new version of pom1pom_1pom1​ (pom1′′pom''_1pom1′′​) and (ii) the generation of a new prediction y^1,c3\hat{y}_{1,c3}y^​1,c3​ for case c3, which is still running at t8. Finally, note that a prediction can only be evaluated when the corresponding label becomes available. In the example, the predictions generated for all the prefixes of case 3 cannot be evaluated because the label of case 3 has yet to be received at t8.
